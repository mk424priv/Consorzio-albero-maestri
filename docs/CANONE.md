## 0. Філософія (5 принципів)

1. **Все крутиться навколо ЗАПИСУ РОБОТИ.** Заходжу → одразу «Новий запис».
   Як у записнику: зафіксувати те, що зробив, або те, що треба зробити.
2. **Три рівні сутностей:** `Cliente` (клієнт), `Collaboratore` (працівник;
   за замовчуванням — це Я), `Lavoro` (робота/запис). Більше нічого в ядрі.
3. **Прогресивне розкриття.** На екрані видно ТІЛЬКИ те, що стосується
   поточного вибору. Додаткове — відкривається; деталі — розгортаються;
   нічого не змішується. Порожнього шуму немає.
4. **Два часові режими кардинально різні:** `FATTO` (факт/історія) та
   `DA FARE` (план). Вони міняють сенс полів, кнопок, підсумків.
5. **Один лінійний сценарій з гілками.** Кожен крок логічно веде до
   наступного. На кожне «а що якщо…» є канонічна відповідь (розділ 9).

## 1. Модель даних (ядро)

Спрощуємо до мінімуму. (Технічно мапиться на наявний store, але концептуально
це канон.)

### 1.1 Cliente
| Поле | Тип | Призначення |
|---|---|---|
| id | string | — |
| nome | string | імʼя/назва (може бути лише імʼя) |
| tariffaOraria | number? | €/год за замовчуванням (для режиму «Ore») |
| contatti | {telefono?, luogo?, note?} | необовʼязково |
| creatoIl | date | — |

### 1.2 Collaboratore (працівник)
| Поле | Тип | Призначення |
|---|---|---|
| id | string | — |
| nome | string | імʼя |
| tariffaOraria | number? | €/год — **вартість** його роботи (для мене це витрата) |
| ruolo | "io" \| "collaboratore" | `io` = Я (титульний) |
| attivo | bool | — |

**Канон:** завжди існує один `io`. Його години **не створюють витрати** (це
мій прибуток). Години інших — це витрата (compenso). Це базове припущення;
можна вимкнути в налаштуваннях запису (галочка «і мою роботу рахувати як
витрату»).

### 1.3 Lavoro (запис роботи) — центральна сутність
| Поле | Тип | Призначення |
|---|---|---|
| id | string | — |
| clienteId | string | до кого привʼязано |
| titolo | string? | автозаповнюється («Intervento {data}») |
| **fase** | `"fatto"` \| `"da_fare"` | часовий режим (розділ 2.A) |
| **modo** | `"preventivo"` \| `"ore"` | режим обліку (розділ 2.B) |
| **conteggio** | `"per_giorni"` \| `"totale"` | спосіб лічби (тільки для `ore`) |
| data | date | головна дата (для сортування/групування) |
| periodo | {dal, al}? | для `preventivo`: за який період взято суму |
| prezzo | number? | для `preventivo`: погоджена сума |
| righe | RigaGiorno[] | для `per_giorni`: рядки по днях |
| partecipanti | Partecipante[] | хто працював (мін. `io`) |
| spese | Spesa[] | витрати по цій роботі |
| incassato | number | скільки вже оплачено |
| note | string? | — |
| creatoIl | date | — |

### 1.4 Вкладені структури
```
Partecipante = { collaboratoreId, oreTotale }      // для conteggio="totale"
RigaGiorno   = { data, ore: { [collaboratoreId]: number } }  // для "per_giorni"
Spesa        = { id, descrizione, importo }
```

> `ore` працівника завжди можна задати на запис (totale) або по днях
> (per_giorni). Сума по днях = підсумкові години працівника.

### 1.5 Похідні величини (рахуються, не зберігаються)
- `oreTotali` = Σ годин усіх учасників (або по днях).
- `lordo` (брутто):
  - `modo="preventivo"` → `prezzo`.
  - `modo="ore"` → `Σ (oreCliente) × cliente.tariffaOraria`. *(години Я + інші,
    бо клієнту виставляється повний час; за замовчуванням Я рахую за тарифом
    клієнта; але див. 9.7).*
- `costoCollaboratori` = Σ (oreCollab × collab.tariffaOraria) для всіх `≠ io`.
- `speseTotali` = Σ spese.importo.
- `netto` = `lordo − speseTotali − costoCollaboratori`.
- `daIncassare` = `lordo − incassato` (≥0).
- `statoPagamento` = «не оплачено» / «частково» / «оплачено».

## 2. Режими (повна матриця)

### A. Часова фаза — `fase`
| | FATTO (факт) | DA FARE (план) |
|---|---|---|
| Сенс | вже зроблено, фіксую історію | заплановано, ще не зроблено |
| Дата за замовч. | сьогодні | найближча майбутня / обрана |
| Підсумки | реальні години/гроші | очікувані/оцінка |
| Інкасо | можна одразу записати оплату | приховано (нічого ще не взято) |
| Вигляд картки | звичайна | пунктирна рамка + бейдж «план» |
| Дія-конверсія | — | кнопка «Позначити як зроблено» → стає FATTO |

### B. Режим обліку — `modo`
| | PREVENTIVO (за суму) | ORE (за години) |
|---|---|---|
| Головне поле | `prezzo` (сума, яку взяв) | години |
| Додатково | `periodo {dal,al}` — за який період | вибір `conteggio` |
| Брутто | = prezzo | = години × тариф клієнта |
| Тариф клієнта | не потрібен | потрібен (якщо нема → модалка) |
| Години | необовʼязково (лише для costo) | обовʼязково |

### C. Спосіб лічби годин — `conteggio` (тільки `modo="ore"`)
| | PER_GIORNI (по днях) | TOTALE (загально) |
|---|---|---|
| Що задаю | рядки: дата + години (на учасника) | одне поле годин (на учасника) |
| Підсумок | сума по рядках | пряме значення |
| Коли | багатоденна робота, хочу деталізацію | один раз, не важливі дні |

**Канонічні комбінації (4 основні × лічба):**
1. `da_fare + preventivo` — план із погодженою ціною й датою.
2. `da_fare + ore` — план з оцінкою годин.
3. `fatto + preventivo` — зроблено, взяв суму за період.
4. `fatto + ore (totale)` — зроблено, X годин разом.
5. `fatto + ore (per_giorni)` — зроблено, рядки по днях, сумується.

## 3. ГОЛОВНИЙ СЦЕНАРІЙ — створення запису

Покроковий майстер з прогресивним розкриттям. Кожен крок відкриває наступний.
Усе на одному екрані, секції розгортаються згори вниз.

```
[ Nuovo intervento ]   ← головна кнопка (центр)
        │
  ┌─────▼─────────────────────────────────────────────┐
  │ КРОК 1 — Фаза (великий перемикач)                  │
  │   ( ✅ FATTO )   ( 🕓 DA FARE )                      │
  └─────┬──────────────────────────────────────────────┘
        │ обрано → задає тон усьому
  ┌─────▼──────────────────────────────────────────────┐
  │ КРОК 2 — Дата                                       │
  │   chip «Oggi» + дата. (FATTO=сьогодні, PIANO=далі)  │
  └─────┬──────────────────────────────────────────────┘
  ┌─────▼──────────────────────────────────────────────┐
  │ КРОК 3 — Cliente                                    │
  │   [пошук/нове імʼя]  або  [📋 Scegli dalla lista]    │
  │   • вибір зі списку → оверлей списку клієнтів        │
  │   • нове імʼя → при переході МОДАЛКА «Imposta tariffa│
  │     per questo cliente» (€/год) — один раз            │
  └─────┬──────────────────────────────────────────────┘
  ┌─────▼──────────────────────────────────────────────┐
  │ КРОК 4 — Modo (перемикач)                            │
  │   ( 💶 PREVENTIVO )      ( ⏱ ORE )                   │
  │                                                     │
  │  PREVENTIVO ▼                 ORE ▼                  │
  │   • prezzo (€)               • conteggio:           │
  │   • periodo dal–al            (Totale)(Per giorni)   │
  │                              • години (rotella/step) │
  │                              • [per_giorni]: рядки   │
  │                                дата+години (+додати) │
  └─────┬──────────────────────────────────────────────┘
  ┌─────▼──────────────────────────────────────────────┐
  │ КРОК 5 — Collaboratori (необовʼязково)              │
  │   default: solo io.   [ + Aggiungi persona ]        │
  │   на кожного: вибір/створення + години               │
  │   (per_giorni → години по кожному дню)              │
  └─────┬──────────────────────────────────────────────┘
  ┌─────▼──────────────────────────────────────────────┐
  │ КРОК 6 — Spese (необовʼязково, згорнуто)            │
  │   [ + Aggiungi spesa ]  рядок: descrizione + €      │
  └─────┬──────────────────────────────────────────────┘
  ┌─────▼──────────────────────────────────────────────┐
  │ ПІДСУМОК (липкий внизу, оновлюється в реальному часі)│
  │   Lordo … − Spese … − Collab … = NETTO …            │
  │   [ FATTO: + Segna incasso ]                         │
  │   [ ✅ Salva intervento ]                            │
  └─────────────────────────────────────────────────────┘
```

**Правила розкриття у майстрі:**
- Крок N+1 зʼявляється тільки після заповнення/вибору кроку N (плавно).
- Поля `prezzo/periodo` видно лише для `preventivo`; поля годин/днів — лише
  для `ore`. Перемикання `modo` ховає/показує відповідні блоки.
- `Collaboratori` і `Spese` — згорнуті за замовчуванням (одна кнопка), щоб не
  лякати. Розгортаються на дотик.
- Підсумок завжди внизу, рахується наживо.

## 4. ГОЛОВНА СТОРІНКА (центр) — «Записи»

Це календар-записник: список карток робіт за місяць.

- **Групування:** за днями. Заголовок групи = `дата + назва дня`
  (es. «Lunedì 23 giugno»).
- **Місяць:** навігація ‹ Червень 2026 › (за замовчуванням — поточний).
- **Фільтр-перемикач зверху:** `Tutti / Fatto / Da fare`.
- **Компактна картка роботи:**
  ```
  ▎[cliente]                          [badge fase]
    ⏱ Xh   💶 incassato Y / atteso Z   • netto N
    [modo icon]  [n collaboratori]
  ```
  - колірний хребет = fase (план/факт);
  - праворуч стан грошей (residuo / saldato);
  - тап по картці → деталь роботи (розділ 7).
- **Плаваючий бейдж (стиль Telegram):** показує найближчий `da_fare`
  («Prossimo: martedì · Rossi») — тап прокручує/відкриває його.
- **Порожній місяць:** велика підказка «+ Crea il primo intervento».

## 5. ЛІВА СТОРІНКА — «Dashboard» (звідусіль один погляд)

Один екран, тільки релевантні агрегати, **нічого не змішано**, з перемикачем
проєкції.

- **Перемикач проєкції:** `Generale / Per cliente / Per collaboratore`.
- **Період:** місяць / усе (та можливо квартал).

| Проєкція | Що показує |
|---|---|
| Generale | Lordo, Spese, Compensi, **Netto**, Da incassare, годинник тренду |
| Per cliente | список клієнтів: incassato, da incassare, ore, netto → провал |
| Per collaboratore | список працівників: ore, dovuto, pagato, saldo → провал |

- Кожен рядок **клікабельний** → провалюється у відповідний профіль
  (картка клієнта/працівника) з його записами.
- Дані не дублюються між проєкціями — кожна відповідає на одне питання.

## 6. ПРАВА СТОРІНКА — «Anagrafiche» (хто є хто)

Список сутностей + «хто, що, кому, коли, скільки».

- **Вкладки:** `Clienti / Collaboratori`.
- **Clienti:** картки клієнтів (тариф, к-сть робіт, борг). Дії: створити,
  редагувати, відкрити профіль (усі його роботи, гроші, історія).
- **Collaboratori:** картки працівників (тариф, години, скільки винен). Дії:
  створити, редагувати, відкрити профіль (його роботи й компенси).
- Фільтри/пошук. CRUD тут (а не в майстрі — майстер лише швидко створює «на
  льоту»).

## 7. ДЕТАЛЬ ЗАПИСУ (повний екран роботи)

Куди провалююся з картки. Усі дані й налаштування, згруповані по сенсу.

- **Шапка:** клієнт, дата, fase (перемикач-степер), modo.
- **Економіка:** Lordo / Spese / Collab / **Netto**; смужка інкасо;
  кнопка `Incassa` (для fatto) з вибором суми (часткова оплата дозволена).
- **Години:** totale або таблиця по днях (рядки редагуються інлайн);
  розбивка по учасниках.
- **Collaboratori:** список з годинами та вартістю; додати/прибрати.
- **Spese:** список; додати/редагувати/видалити інлайн.
- **Periodo (preventivo):** dal–al.
- **Дії:** Segna come fatto (якщо план), Duplica, Elimina, Note.
- Усе **копіюється/редагується/видаляється тут же** (як домовились раніше).

## 9. КАНОНІЧНІ ВІДПОВІДІ («що якщо…»)

1. **Створюю без клієнта?** Пишу імʼя → клієнт створюється на льоту →
   зʼявляється модалка «Imposta tariffa» (можна пропустити для preventivo,
   обовʼязково для ore).
2. **Клієнт без тарифу, а обрав ore?** Модалка тарифу блокує продовження
   режиму ore (бо без тарифу не порахувати). Preventivo — працює без тарифу.
3. **Preventivo + period (dal–al)?** Сума стосується періоду (напр.
   «manutenzione 100€ giugno»). Використовується для звітів/повторюваності;
   на netto впливає лише `prezzo`.
4. **Ore, per_giorni — як сумується?** Σ годин по рядках = години запису. Якщо
   кілька учасників — кожен має свої години в кожному дні.
5. **Da fare → робота сталась?** Кнопка «Segna come fatto»: fase→fatto, дата
   може оновитись на фактичну, відкривається інкасо.
6. **Інша людина працювала менше/більше?** На кожного учасника — свої години
   (і по днях, якщо per_giorni). Вартість рахується індивідуально за тарифом.
7. **Мою роботу рахувати як витрату?** За замовчуванням НІ (io = прибуток).
   Галочка в записі «conta anche le mie ore come costo» → тоді io теж costo.
8. **Часткова оплата?** `incassato` може бути < `lordo`; статус «частково»,
   residuo = lordo − incassato. Не можна інкасувати більше lordo.
9. **Видаляю клієнта/працівника, що з записами?** М'яке приховування
   (архів), записи лишаються з позначкою «архівний». Нічого не зникає.
10. **Змінив modo після введення даних?** Дані попереднього режиму
    зберігаються «в тіні» (не видаляються), показуються поля нового; при
    поверненні — відновлюються.
11. **Робота без години і без ціни (просто факт)?** Дозволено: lordo=0,
    лишається як подія в історії клієнта.
12. **Кілька робіт в один день?** Кожна — окрема картка; групуються під одним
    заголовком дня.
13. **Майбутня дата для FATTO?** Попередження (факт у майбутньому нелогічний),
    але не блокується.

## 11. ЕТАПИ РЕАЛІЗАЦІЇ (після затвердження)

1. **Ядро даних + майстер створення** (розділи 1–3) — серце.
2. **Головна сторінка записів** (розділ 4) + плаваючий бейдж.
3. **Деталь запису** (розділ 7) з повним редагуванням.
4. **Dashboard** (розділ 5) з проєкціями.
5. **Anagrafiche** (розділ 6) + профілі.
6. **Прогресивне розкриття/полірування** (розділ 0.3, мікроанімації, стани).
