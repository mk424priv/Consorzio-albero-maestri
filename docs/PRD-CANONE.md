## 0. Глосарій

- **Intervento / Lavoro / Запис** — одна одиниця роботи. Центральна сутність.
- **Cliente** — для кого роблю. Має тариф (€/год) як замовчування.
- **Collaboratore** — хто робить. `io` (Я, титульний) завжди є.
- **Fase** — `fatto` (факт/історія) або `da_fare` (план).
- **Modo** — `preventivo` (за суму) або `ore` (за години).
- **Conteggio** — `totale` або `per_giorni` (рядки по днях).
- **Lordo** — брутто (виставлено клієнту). **Netto** — після витрат і компенсів.
- **Знімок (snapshot)** — зафіксоване значення тарифу всередині запису, щоб
  історія не «попливла» при зміні тарифу клієнта/працівника згодом.
- **Анкета (Anagrafica)** — профіль клієнта/працівника (права зона).

## 1. Персона і Jobs-To-Be-Done

**Персона:** садівник/деревороб на Ельбі. Працює в полі, телефон у руці,
брудні рукавиці, мало часу, сонце. Часто працює сам, іноді з 1–2 помічниками.
Бере оплату або «за роботу» (домовлена сума), або «за години».

**Головні JTBD (у порядку частоти):**
1. «Щойно закінчив у клієнта — хочу за 20 секунд зафіксувати: хто, скільки
   годин/яка сума, які витрати.» → **FATTO-запис**.
2. «Планую завтра поїхати до клієнта.» → **DA FARE-запис**.
3. «Скільки мені винні? Скільки я зарекомендував цього місяця чистими?»
   → **Dashboard**.
4. «Скільки годин зробив Лука цього місяця, скільки я йому винен?»
   → **Dashboard / Anagrafiche**.
5. «Покажи історію по цьому клієнту.» → **профіль клієнта**.

**Анти-цілі:** не бухгалтерія, не CRM, не складна форма. Швидкість і ясність
важливіші за повноту.

## 3. Канон даних і знімки тарифів

### 3.1 Cliente
```
Cliente {
  id
  nome                  // обовʼязкове по факту, але авто-дефолт «Cliente»
  tariffaOraria?: €/h   // замовчування для режиму ore
  telefono?, luogo?, note?
  attivo: bool          // false = архівований (не видалений)
  creatoIl
}
```

### 3.2 Collaboratore
```
Collaboratore {
  id
  nome
  tariffaOraria?: €/h    // ВАРТІСТЬ його роботи (моя витрата)
  ruolo: "io" | "collaboratore"
  attivo: bool
  creatoIl
}
```
Канон: рівно один `io`. `io.tariffaOraria` за замовчуванням = 0 для розрахунку
витрат (моя праця = прибуток), але може мати власний тариф для статистики
(див. 4.4 і 22-Q).

### 3.3 Lavoro
```
Lavoro {
  id
  clienteId
  titolo?                         // авто: «Intervento {data}»
  fase: "fatto" | "da_fare"
  modo: "preventivo" | "ore"
  conteggio: "totale" | "per_giorni"   // лише для ore
  data                             // головна (для групування)
  periodo?: { dal, al }            // для preventivo (мітка періоду)
  prezzo?: number                  // для preventivo
  // ЗНІМКИ тарифів на момент створення:
  tariffaClienteSnapshot?: number  // для ore-розрахунку (стабільна історія)
  partecipanti: Partecipante[]     // мін. [io]
  righe: RigaGiorno[]              // для per_giorni
  spese: Spesa[]
  incassato: number                // 0..lordo
  contaMieOreComeCosto: bool       // дефолт false
  note?
  creatoIl
}

Partecipante {
  collaboratoreId
  tariffaSnapshot: number          // €/h зафіксований при додаванні
  oreTotale?: number               // для conteggio=totale
}
RigaGiorno {
  data
  ore: { [collaboratoreId]: number }   // години кожного учасника в цей день
}
Spesa { id, descrizione, importo }
```

### 3.4 Канон знімків (важливо!)
- При **створенні/збереженні** запису з `modo=ore` фіксуємо
  `tariffaClienteSnapshot = cliente.tariffaOraria` (на той момент).
- При **додаванні учасника** фіксуємо `tariffaSnapshot = collaboratore.tariffaOraria`.
- Подальша зміна тарифу клієнта/працівника **не змінює** старі записи
  (історія стабільна). Але в деталі є кнопка **«Aggiorna tariffe a quelle
  attuali»** (ручний перерахунок за бажанням).
- Новий запис завжди бере поточні тарифи.

### 3.5 Інваріанти (завжди істинні)
- У кожного `Lavoro` мінімум один учасник (`io`), навіть якщо 0 годин.
- `0 ≤ incassato ≤ lordo`.
- `fase=da_fare` ⇒ `incassato = 0` (план нічого не отримав).
- `modo=preventivo` ⇒ ігнорує тариф клієнта для lordo (lordo = prezzo).
- `conteggio=per_giorni` ⇒ години учасника = Σ по `righe`.

## 4. Канон розрахунків (єдине джерело правди)

### 4.1 Години
- `oreParticipante(p)` = `conteggio=totale ? p.oreTotale : Σ righe.ore[p.id]`.
- `oreTotali` = Σ по всіх учасниках.
- `oreCliente` = години, що виставляються клієнту = `oreTotali`
  (усі, хто працював, — це час, за який клієнт платить).

### 4.2 Lordo (брутто, виставлено клієнту)
- `modo=preventivo` → `lordo = prezzo ?? 0`.
- `modo=ore` → `lordo = oreCliente × tariffaClienteSnapshot`.

### 4.3 Costo collaboratori (мої витрати на чужу працю)
- `costo = Σ (oreParticipante(p) × p.tariffaSnapshot)` для всіх `p ≠ io`.
- Якщо `contaMieOreComeCosto=true` → додаємо й `io`.

### 4.4 Spese / Netto / Інкасо
- `speseTotali = Σ spese.importo`.
- `netto = lordo − speseTotali − costo`.
- `daIncassare = max(0, lordo − incassato)`.
- `statoPagamento`: `incassato=0`→«non pagato»; `0<incassato<lordo`→«parziale»;
  `incassato≥lordo`→«pagato».
- `margine %` (опц.) = `netto / lordo` якщо `lordo>0`.

### 4.5 Агрегації (для Dashboard/профілів)
- Per cliente: Σ lordo, Σ incassato, Σ daIncassare, Σ ore, Σ netto.
- Per collaboratore: Σ годин, `dovuto = Σ (ore × tariffaSnapshot)`,
  `pagato` (з компенсів), `saldo = dovuto − pagato`.
- Generale (місяць): Σ lordo, Σ spese, Σ costo, **Netto**, Σ daIncassare.
- Розрахунки за **датою запису** (для preventivo з periodo — за `data`; авто-
  розбивка по місяцях — відкрите питання 22-Q5).

## 5. Життєвий цикл CLIENTE (повні сценарії + ефекти)

Це те, чого бракувало. Розписуємо **усі точки створення**, **модалку тарифу**,
**ефекти на кожен екран/функцію** і **редагування/архів**.

### 5.1 Три точки створення
| # | Звідки | Як | Поля одразу |
|---|---|---|---|
| A | **Майстер запису** (вводжу нове імʼя в полі «Cliente») | inline, найшвидше | лише `nome` + (модалка тарифу) |
| B | **Список вибору** (оверлей у майстрі) → «+ Nuovo cliente» | швидка форма | `nome`, `tariffa?`, контакти? |
| C | **Anagrafiche → Clienti → +** | повна форма | усе |

### 5.2 Сценарій A (inline у майстрі) — детально
```
Поле «Cliente»: вводжу «Rossi» → нема збігу у списку
   → під полем підказка: «Nuovo cliente: «Rossi» (verrà creato)»
Натискаю далі / обираю modo:
   ┌ ЯКЩО modo=ore:
   │   МОДАЛКА «Imposta tariffa per Rossi» (€/h) — ОБОВʼЯЗКОВА
   │     [поле €/h] [Salva]  [Salta (lascia 0)]
   │     • Salva → клієнт створюється з тарифом, snapshot у запис
   │     • Salta → тариф 0 → lordo=0, попередження «lordo a 0»
   └ ЯКЩО modo=preventivo:
       тариф НЕ потрібен → модалка НЕ зʼявляється (тариф можна задати пізніше)
Клієнт створюється у мить збереження запису (не раніше),
   АЛЕ для UX показуємо як «обраний» одразу (оптимістично).
```
**Канон:** клієнт фізично створюється при **Salva intervento** (щоб
скасування майстра не лишало «сміттєвих» клієнтів). Виняток: якщо модалку
тарифу заповнено — створюємо одразу (бо є осмислені дані).

### 5.3 Модалка тарифу — повна логіка
- **Тригер:** обрано `modo=ore` І у клієнта `tariffaOraria` відсутній (новий
  або старий без тарифу).
- **Поведінка:** блокує продовження ore-гілки, доки не задано тариф або
  натиснуто «Salta».
- **Salta:** тариф лишається порожнім; запис дозволений, але `lordo=0` і badge
  «⚠ tariffa mancante» у деталі; у профілі клієнта банер «Imposta tariffa».
- **Не зʼявляється** для `preventivo` (там сума вводиться вручну).
- **Повторне використання:** наступний ore-запис цього клієнта вже НЕ показує
  модалку (тариф є). Якщо тариф змінився — діє знімок (4 / 3.4).

### 5.4 Ефекти створення клієнта (де і що зʼявляється)
| Місце | Ефект одразу після створення |
|---|---|
| Поле «Cliente» у майстрі | стає обраним (чип з імʼям) |
| Список вибору клієнта | новий рядок угорі (нещодавні) |
| **Anagrafiche → Clienti** | нова картка (тариф або «—», 0 lavori, 0 borgo) |
| **Dashboard → Per cliente** | новий рядок з нулями (поки нема робіт) |
| Пошук (верх) | знаходиться за імʼям |
| Профіль клієнта | створюється, усі секції — empty-state |
| Майбутні ore-записи | беруть його тариф автоматично |

### 5.5 Редагування клієнта
- Звідки: профіль клієнта (кнопка Modifica), Anagrafiche (кнопка на картці),
  деталь запису (тап на імені клієнта → «Modifica cliente»).
- Зміна `tariffaOraria`:
  - **НЕ** змінює старі записи (знімки).
  - Застосовується до **нових** записів.
  - У старому записі — кнопка «Aggiorna tariffa» (ручний перерахунок).
- Зміна `nome`: оновлюється всюди (імʼя не знімкуємо).

### 5.6 Архівація клієнта (замість видалення)
- Дія «Archivia» (не «Elimina» за замовч.).
- Ефекти: зникає зі списків вибору й активних агрегатів; його записи лишаються
  з поміткою «cliente archiviato»; у профілі — банер «archiviato» + «Ripristina».
- Жорстке «Elimina» — лише з подвійним підтвердженням; видаляє клієнта і його
  записи (попередження зі списком того, що зникне).

### 5.7 Дублікати
- При вводі імені, що точно збігається з наявним активним клієнтом —
  підказка «Esiste già: Rossi — selezionalo?» (щоб не плодити дублі).

## 6. Життєвий цикл COLLABORATORE

### 6.1 Точки створення
| # | Звідки | Поля |
|---|---|---|
| A | Майстер → секція Collaboratori → «+ Aggiungi» → нове імʼя | `nome` + модалка тарифу (вартість) |
| B | Anagrafiche → Collaboratori → + | повна форма |

### 6.2 Модалка тарифу для працівника
- Тригер: додаю працівника (≠ io) без тарифу.
- Сенс: це **вартість його години** (моя витрата). Без неї `costo` від нього =0
  (попередження «collaboratore senza tariffa: costo 0»).
- `Salta` дозволено (тоді його праця безкоштовна в розрахунку).

### 6.3 «io» (Я)
- Завжди присутній, не видаляється, не архівується.
- За замовч. у кожному записі — учасник `io` з 0 годин (заповнюю фактичні).
- `io` за замовчуванням не дає costo (мій прибуток). Галочка в записі змінює.

### 6.4 Ефекти створення працівника
- Зʼявляється у чипах вибору в майстрі, у Anagrafiche → Collaboratori, у
  Dashboard → Per collaboratore (з нулями), у профілі (empty).

### 6.5 Редагування / архів — як у клієнта (5.5–5.6), зі знімками тарифу.

## 7. Життєвий цикл LAVORO (стейт-машина)

### 7.1 Стани та переходи фази
```
        (створення)
            │
      ┌─────┴─────┐
      ▼           ▼
   DA FARE  ──►  FATTO        // «Segna come fatto»
      ▲           │
      └───────────┘           // «Riporta a da fare» (рідко, з підтвердж.)
```
- `da_fare → fatto`: відкриває інкасо; дата може оновитись на фактичну
  (запит «data effettiva?»); години-оцінка стають фактичними (можна правити).
- `fatto → da_fare`: скидає `incassato` (з підтвердженням), ховає інкасо.

### 7.2 Підстани оплати (для fatto)
`non pagato → parziale → pagato` (за `incassato`; див. 4.4). Зворотно —
редагуванням `incassato`.

### 7.3 Дії над записом
Crea · Apri (деталь) · Modifica (інлайн у деталі) · Segna fatto/da_fare ·
Incassa (часткова/повна) · Duplica (копія на нову дату) · Archivia · Elimina.

### 7.4 Duplica (сценарій)
- «Duplica» створює новий `da_fare` з тими ж клієнтом/учасниками/тарифами,
  `incassato=0`, дата = сьогодні+? (питає). Корисно для повторюваних робіт.

## 8. Майстер запису — поле-за-полем (повна гілковість)

> Один екран, секції розкриваються згори вниз. Кожне поле має: вид за замовч.,
> стани, валідацію, що відкриває/ховає.

### Крок 1 — Fase (обовʼязково, перше)
- Вигляд: два великі сегменти `✅ Fatto` / `🕓 Da fare`. Дефолт = `Fatto`.
- Ефект: задає дефолт дати, наявність інкасо, тон карток.

### Крок 2 — Data
- Вигляд: чип `Oggi` + нативний date. Дефолт: fatto=сьогодні; da_fare=сьогодні
  (можна вперед).
- Валідація: fatto з майбутньою датою → мʼяке попередження (не блок).

### Крок 3 — Cliente
- Вигляд: поле пошуку з підказками + кнопка «📋 Lista» (оверлей).
- Стани: (a) порожньо → показ «recenti» чипами; (b) ввід → фільтр; (c) збіг →
  вибір; (d) нове імʼя → підказка «verrà creato» + (модалка тарифу за умов 5.3).
- Обрано → чип з імʼям + ✕ (змінити).
- Валідація: якщо порожньо і не введено імені при Salva → «Indica un cliente».

### Крок 4 — Modo
- Вигляд: сегменти `💶 Preventivo` / `⏱ Ore`. Дефолт: останній використаний
  (памʼять) або `Ore`.
- **Preventivo розкриває:**
  - `prezzo` (€, велике поле, суфікс «presi»).
  - `periodo {dal,al}` (згорнуто, опц.): «per quale periodo» (напр. місяць).
- **Ore розкриває:**
  - `conteggio`: сегменти `Totale` / `Per giorni` (дефолт `Totale`).
  - `Totale` → одне поле годин (степер/рулетка) для кожного учасника
    (мінімум для `io`).
  - `Per giorni` → список рядків: `[дата] [години io] (+ інші учасники)`;
    кнопка «+ Aggiungi giorno»; внизу авто-сума.
  - Якщо тариф клієнта відсутній → модалка тарифу (5.3).
- Перемикання modo: дані іншого режиму зберігаються «в тіні», не зникають.

### Крок 5 — Collaboratori (згорнуто; «io» вже всередині)
- Рядок `io` завжди; інші — «+ Aggiungi persona».
- На кожного: чип-вибір/створення + години (totale або по днях).
- Якщо per_giorni: години учасника задаються в кожному рядку дня.

### Крок 6 — Spese (згорнуто)
- «+ Aggiungi spesa» → рядок `descrizione + €`. Сума показується.

### Крок 7 — Підсумок (липкий)
- Рядки: `Lordo … − Spese … − Collab … = NETTO …`.
- Для fatto: блок «Incasso» — «quanto hai preso?» (0..lordo), або «Saldato».
- Кнопка `✅ Salva intervento` + міні-резюме (`Xh · €Y`).
- Після Salva: festa, тост «salvato e smistato», скидання, повернення у Центр,
  нова картка зʼявляється у потрібному дні.

---

## 9. Екран ЦЕНТР — «Записи» (повний)

- **Шапка:** ‹ Місяць Рік › (навігація), перемикач `Tutti/Fatto/Da fare`.
- **Тіло:** групи по днях (заголовок `дата + назва дня`), всередині — картки.
- **Картка (компакт):**
  - хребет кольору = fase; іконка modo;
  - `[Cliente]`  `[badge fase]`;
  - `⏱ Xh`  ·  `💶 incassato/atteso`  ·  `netto`;
  - якщо da_fare: пунктир + «pianificato»; якщо residuo>0: бейдж суми.
  - тап → деталь.
- **Плаваючий бейдж «Prossimo»** (стиль Telegram, правий низ): найближчий
  `da_fare` → тап прокручує до нього / відкриває.
- **Empty-state місяця:** ілюстрація + «+ Crea il primo intervento».
- **Швидкі дії на картці (опц., свайп):** Fatto✓, Incassa, Duplica.

## 10. Екран DASHBOARD (ліво) — повний

- **Перемикач проєкції:** `Generale / Per cliente / Per collaboratore`
  (нічого не змішано).
- **Період:** ‹ Місяць › + чип «Tutto».
- **Generale:** великі плитки `Lordo / Spese / Compensi / **Netto**` +
  `Da incassare`; маленький графік тренду netto по місяцях.
- **Per cliente:** список (incassato, da incassare, ore, netto) з сортуванням
  (борг/нетто/імʼя); рядок → профіль клієнта.
- **Per collaboratore:** список (ore, dovuto, pagato, saldo); рядок → профіль.
- **Принцип:** кожна проєкція = відповідь на одне питання; дані не дублюються.

---

## 11. Екран ANAGRAFICHE (право) — повний

- **Вкладки:** `Clienti / Collaboratori`.
- **Clienti:** пошук + список карток (імʼя, тариф, к-сть робіт, борг). Дії:
  `+ Nuovo`, Modifica, Apri profilo, Archivia.
- **Collaboratori:** список (імʼя, тариф-вартість, години міс., saldo). Дії як
  вище + Paga (компенс).
- **Профіль (клієнт/працівник):** шапка + агрегати + список повʼязаних записів
  (тап → деталь запису). «Хто/що/кому/коли/скільки».

## 12. ДЕТАЛЬ ЗАПИСУ — повний

- **Шапка:** Cliente (тап→профіль/змінити), Data, `fase`-степер, `modo`-мітка.
- **Економіка:** Lordo/Spese/Collab/**Netto**; смуга інкасо; `Incassa` (для
  fatto, часткова дозволена; не більше lordo).
- **Години:** totale (поля) або таблиця per_giorni (інлайн-редаг.); розбивка
  по учасниках з годинами й вартістю.
- **Collaboratori:** список +/−, кожен з годинами/тарифом-знімком; «Aggiorna
  tariffe» (перерахунок за поточними).
- **Spese:** інлайн список (додати/правити/видалити).
- **Periodo (preventivo):** dal–al (редаг.).
- **Дії:** Segna fatto/da fare, Duplica, Archivia, Elimina, Note.
- Усе копіюється/редагується/видаляється тут же.

## 14. Прогресивне розкриття — точні правила

1. Крок майстра N+1 рендериться лише коли крок N має валідне значення.
2. Поля `prezzo/periodo` існують у DOM лише для `preventivo`; поля годин/днів —
   лише для `ore`. Перемикання — анімований показ/прихов, дані в тіні.
3. `Collaboratori` і `Spese` згорнуті (1 кнопка) доки користувач не відкриє.
4. Модалки (тариф, список вибору) — поверх, із затемненням, фокус усередині.
5. У картках показуємо лише непорожнє (нема spese → не показуємо рядок spese).
6. «Розгортай коли треба»: деталі дня/учасника — акордеоном.
7. Жодних «мертвих» кнопок: дія зʼявляється лише коли застосовна (напр.
   `Incassa` лише для fatto з residuo>0).

## 15. Валідації та обмеження (канон)

- Імена/титули: ніколи не блокують (авто-дефолт); дублі — мʼяка підказка.
- `prezzo`, `ore`, `spese.importo`: ≥ 0; порожнє = 0; режим не зберігає
  порожніх «сміттєвих» рядків.
- `ore` на день/учасника: 0..24 (понад — попередження).
- `incassato`: 0..lordo (не можна понад lordo; не можна <0).
- `da_fare` ⇒ incassato заблоковано (0).
- `ore`-режим без тарифу клієнта ⇒ модалка (або явний Salta з lordo=0).
- Дата fatto у майбутньому ⇒ попередження, не блок.
- Видалення ⇒ підтвердження зі списком наслідків; архів — без підтвердження.

## 16. Каталог станів (empty / edge / error)

- **Центр без записів (місяць):** «Nessun intervento a giugno» + кнопка.
- **Центр узагалі порожній (перший запуск):** онбординг-картка «Crea il
  primo intervento» + 1 рядок пояснення.
- **Клієнт без робіт (профіль):** empty по кожній секції.
- **Клієнт без тарифу + ore:** банер/модалка.
- **Працівник без тарифу:** попередження «costo 0».
- **Lavoro з lordo 0:** дозволено (подія-факт), badge «senza importo».
- **Da fare у минулому (прострочений план):** badge «in ritardo» + у бейджі
  «prossimo» показуємо найперший прострочений.
- **Часткова оплата:** badge «parziale» + residuo.
- **Архівований клієнт у старому записі:** мітка, але запис читається.
- **Конфлікт дати при per_giorni (два рядки на один день):** обʼєднати чи

години 6→Incassa 0 (ще не взяв). Тепер у фактичних сумах.

**S6. Часткова оплата.** Деталь fatto, lordo 200, Incassa 120 → parziale,

**S9. Повторювана робота.** Деталь→Duplica→нова da_fare на наступний тиждень,
