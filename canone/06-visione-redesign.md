# Канон 06 — Бачення редизайну: «Officina digitale»

> Єдина UX/UI-концепція поверх брендбуку [`05`](05-brandbook.md). Брендбук дає
> **матеріали** (палітра, шрифти, mesh, компоненти). Цей документ дає **архітектуру
> досвіду**: як усе скомпоновано, як кожен запис, кнопка і спливаюче вікно отримують
> **власну індивідуальність**. Реалізація має йти **далі** за брендбук — додавати
> «прикольні штуки» (WebGL у контексті, індивідуальні модалки, мікро-рух).

---

## 1. Натхнення і характер

Беремо точність neo-banking (Revolut / Robinhood / N26 / Linear) + **живий
WebGL-mesh як фірмовий підпис** + великий редакторський шрифт. Прибираємо «папір».
Новий характер — **«цифрова майстерня»**: інструмент майстра, що працює в зеленому
господарстві Ельби. Чорний верстат, на якому **кожна річ світиться своїм світлом**.

> **Девіз системи:** *«Ogni cosa ha la sua luce»* — кожна сутність і кожен стан
> мають власний колір, форму, рух і світло. Список — не однорідна стрічка, а
> **ієрархія індивідуальних блоків**.

Три відчуття, які має давати застосунок: **точність** (нічого зайвого, усе на місці),
**живість** (mesh дихає, числа перетікають, дотик відгукується), **впізнаваність**
(за кольором/формою одразу ясно, що це і в якому воно стані).

## 2. Архітектура екрана — 4 регістри блоків

Кожен екран читається як **партитура з 4 регістрів** згори донизу:

```
┌─────────────────────────────┐
│  TESTATA   (верхній)        │  ← ідентичність екрана: великий заголовок,
│  hero + numero + controllo  │     число-герой (NumberFlow), контрол контексту,
│                             │     опційно WebGL-mesh смуга. Стискається на скролі.
├─────────────────────────────┤
│  RIEPILOGO (інформаційний)  │  ← дашборд-плитки/тотали/KPI. Найважливіша —
│  stat tiles · KPI · glow    │     з mesh/glow fill. 1-3 плитки.
├─────────────────────────────┤
│  ELENCO    (список)         │  ← записи, згруповані за датою/розділом.
│  sezioni · record per stato │     Кожен запис — картка, чия форма/тон = стан.
│                             │
├─────────────────────────────┤
│  AZIONE    (нижній)         │  ← плаваюча nav + FAB + контекстна дія.
│  nav · FAB · barra azione   │
└─────────────────────────────┘
```

**Правила композиції:**
- **Один герой на екран.** У `TESTATA` лише одне число-герой (баланс/тотал дня).
  Решта — підпорядковані.
- **Дихання важливіше за щільність.** Між регістрами — повітря (роздільники
  `bordo`, відступи). Щільність — лише всередині списку.
- **Ліва кромка = стан.** Усі записи мають кольорову акцент-смугу зліва: миттєве
  зчитування стану навіть периферійним зором.
- **Дія завжди праворуч.** Первинна дія запису (`Riscuoti`, `Paga`) — у правому
  нижньому куті картки; навігаційне «відкрити» — тап по тілу.
- **Скрол стискає `TESTATA`** (sticky shrink): герой меншає, контрол лишається.

## 3. Індивідуальність записів (серце редизайну)

Кожен стан — окремий **матеріал**: форма + колір + щільність + рух + світло.

| Сутність / стан | Форма | Колір / світло | Рух |
|---|---|---|---|
| `lavoro · programmato` | повна картка, **повітряна**; час попереду, AvatarStack виконавців, пунктирний акцент | **blu** ліва смуга + кутовий blu-glow | hover: легке підняття + glow |
| `lavoro · da incassare` | повна картка, **гучна**; ВЕЛИКА сума, біла «Riscuoti», іконка-сигнал | **rosso** смуга + м'який rosso-glow | swipe-вліво → Riscuoti; Riempimento |
| `lavoro · parziale` | повна картка; **дві суми** (incassato/da incassare) | rosso + verde поділ | прогрес-смужка incasso |
| `lavoro · pagato` | **компактний рядок**, спокій; зелене коло-✓ | **verde**, приглушено | без дії; легкий fade-in |
| `cliente · nuovo` | рядок: аватар-ініціали + `codice` + сальдо | нейтральний | — |
| `cliente · storico` | рядок з **mesh-кільцем** навколо аватара + пігулка `Storico` | mesh (blu→verde) ореол | кільце повільно обертається |
| `operaio · io` | рядок, **виділений** (титуляр) | blu акцент | — |
| `operaio · collaboratore` | рядок: аватар + роль + до виплати | rosso якщо борг | — |
| `spesa` | компактний рядок зі стрілкою-вниз | rosso, тонко | — |

**«Старі/лояльні клієнти» (`storico`)** — спеціальна індивідуальність: клієнт із
історією (багато `lavori`, давнє знайомство, високий `valoreFatturabile`) отримує
**mesh-кільце** навколо аватара (живий WebGL/CSS-кон), пігулку `Cliente storico`,
теплішу подачу. Новий клієнт — нейтральний. Так лояльність **видно**.

Градації **відтінків** (та сама родина, різна насиченість):
- `da incassare`: rosso `#ff3b30` повний (терміново) → прострочене ще яскравіше
  (пульс), свіже — спокійніший rosso.
- `pagato`: verde `#00d15e`, але **приглушений** (справу закрито, не кричить).
- `programmato`: blu `#0a58ff`; «сьогодні» — яскравіше, «далеко» — тьмяніше.

## 4. Кнопки — мова дій

Ієрархія заливок (з брендбуку, посилено правилами розташування):
- **Primaria** (біла пігулка) — одна головна дія в контексті; завжди праворуч.
- **Accento** (`bg-blu` / `bg-verde` / `bg-rosso`) — коли дія = подія (підтвердити
  incasso = verde, pagamento = blu, видалити = rosso).
- **Secondaria** (`superficie`) — альтернатива поруч.
- **Utility** (кругла-іконка) — назад/⋯/степери.
- **Action-row** (кругла іконка + підпис) — набір швидких дій під героєм
  (`Crea · Incassa · Dashboard · Altro`).
- **FAB** (біле коло «+») — глобальне створення; **морфить** у вікно створення.

Таб-бар і сегмент — **не кнопки**: скляно-сірий повзунок, лайм/акцент лише на
іконці активного. Це знімає плутанину «де таб, де кнопка».

## 5. Спливаючі вікна — індивідуальна тональність за ієрархією

Не один тип модалки на все. **Кожен sheet має свій верх, колір, радіус і рух**,
що відповідають його **вазі в ієрархії** (danger > azione > dettaglio > info):

| Тип | Призначення | Верх (header) | Тон / світло | Рух |
|---|---|---|---|---|
| `info` | дрібна довідка / вибір | плоский, без mesh | нейтральний | fade + slide-up короткий |
| `dettaglio` | картка-досьє запису | заголовок + аватар | нейтральний, спокійний | spring знизу |
| `azione · incasso` | взяти гроші | **mesh-смуга verde** + число-герой | verde | spring + `Riempimento` + `StampVivo` на підтвердженні |
| `azione · pagamento` | виплатити | **mesh-смуга blu** | blu | spring + Riempimento |
| `creazione` | новий запис (візард) | темний `ossidiana`, індикатор кроків | нейтрально-фокус | FAB→sheet shared-element |
| `pericolo` | видалити/незворотнє | центр-іконка в rosso-колі | **rosso**, різко | швидкий scale-in, без mesh |

«Прикольні штуки» в модалках: **WebGL-mesh смуга у шапці** (тонована під контекст),
**сплеск-частинки** (`StampVivo`) і **Riempimento** на підтвердженні грошей,
**NumberFlow** на сумах, **shared-element** FAB→creazione, **snap-points**
(півекрана / повний) у досьє. Затемнення-підкладка теж тонована (verde/blu/rosso 8%).

## 6. WebGL та рух — фірмовий підпис у контексті

`MeshGradient` (рушій `ogl`) — не лише hero брендбуку, а **контекстний інструмент**:
- **Hero-смуги** в `TESTATA` важливих екранів (Dashboard, Soldi) — тонований mesh.
- **Шапки `azione`-модалок** — mesh під контекст (verde/blu/rosso).
- **Empty-state** — великий mesh + заклик.
- **Свято incasso** — короткий сплеск mesh+частинки при взятті грошей.
- **Кільце `cliente storico`** — mesh-ореол навколо аватара.
- Скрізь: `prefers-reduced-motion` → статичний кадр; нема WebGL → CSS-mesh fallback.

Мова руху: **spring** (живо, не «желе»), `press scale 0.98`, **list stagger reveal**
(картки з'являються каскадом), **swipe-дії** на картках, **collapse** TESTATA,
**NumberFlow** на всіх ключових числах, **shared-element** для FAB і відкриття досьє.

## 7. Принципи, яких тримаємось

1. **Стан видно за 0.2 c** — колір лівої кромки + форма.
2. **Один герой на екран** — решта підпорядкована.
3. **Дія праворуч, навігація — тіло.**
4. **Кожна модалка — свій характер** за ієрархією ваги.
5. **WebGL — у контексті**, не як шпалери.
6. **Дихання > щільність**, окрім середини списку.
7. **Завжди доступно** — контраст, фокус, reduced-motion, тач-таргети ≥44px.
