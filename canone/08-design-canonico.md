# Канон 08 — Канонічний дизайн «Quaderno Vivo»

> **Це фінальна, обов'язкова концепція дизайну.** Консолідує і **переважає** 05–07.
> Кожен новий екран робиться за цим документом. Мета: дизайн **різноманітний, цілісний,
> читабельний, зручний, молодіжний, стильний, в міру мінімалістичний — але креативний**.
> Натхнення: Revolut, сучасні крипто-гаманці (Phantom/Rainbow/Coinbase), fintech 2025.

---

## 1. Бачення і відчуття («the taste»)

Світло денного «верстата майстра» + точність необанку + грайливість крипто-гаманця.
Відчуття: **легке, повітряне, але соковите**; молоде й стильне без крику.

**Три закони смаку (звідси все інше):**
1. **Роздільність — елевацією, контрастом і шаром, НЕ обводками й не glow.** Жодних
   зайвих 1px-обводок, жодних незрозумілих тіней/підсвічень під блоками (це вже не модно).
   Блок «летить» завдяки **сходинці поверхні + м'якій цілеспрямованій тіні + заливці**.
2. **Колір — це дані, не декор.** Насичені акценти кодують **категорію/стан** (заливка,
   іконка, сума), а **не лежать на поверхні кнопок** (правило Revolut). Кнопки — нейтральні.
3. **Різноманіття за характеристикою.** Однакові картки поспіль = нудно. Кожен параметр
   має форму під свою суть: десь велике число, десь сегмент, десь міні-чіп що розгортається,
   десь рядок історії, десь bento-плитка.

## 2. Палітра й ІЄРАРХІЯ відтінків

### Поверхні (світлі, шарами — сходинки елевації)
| Токен | Hex | Роль |
|---|---|---|
| `fondo` | `#eef1f6` | полотно (фон екрана) |
| `superficie` | `#ffffff` | картка (рівень 1) |
| `superficie-2` | `#eef0f5` | вставка / трек / інпут |
| `superficie-3` | `#e3e7ee` | активний сегмент / hover |
| `velo` | `#1418240f` | дуже тонкий розділювач (тільки де треба) |
| `scuro` | `#16181f` | темна поверхня (primary-кнопка, FAB, hero-контраст) |

### Чорнило (текст)
`inchiostro/bianco` `#16181f` (первинний) · `fumo` `#5d6470` (вторинний) · `fumo-2` `#98a0ad` (третинний).

### Акценти (кодування категорій/станів — заливки, не обводки)
| Токен | Hex | Значення |
|---|---|---|
| `blu` | `#3b6ef5` | бренд · інфо · програмовано |
| `verde` | `#15b87a` | позитив · incassato · saldato |
| `corallo` | `#f0584f` | увага · da incassare · уход |
| `ambra` | `#f59331` | попередження · parziale |
| `viola` | `#6a5bd8` | особливе · storico · преміум |
| `ciano` | `#22c3d6` | вторинні дані · veicoli |

### Ієрархія застосування відтінку (3 рівні насиченості)
1. **Велатура (wash)** — 8–12% акценту як **внутрішній градієнт** картки (`linear/radial → білий`).
   Тонко, щоб картка «жила», не кричала. → *колір-кодовані bento-картки*.
2. **Чіп/тінт** — `bg accent/12 + text accent` — для пігулок стану, іконок-кружків, малих міток.
3. **Вівідний градієнт/mesh** — повна заливка hero-блоку (CSS або WebGL mesh), білий текст.
   Тільки **верхній блок/hero** одного екрана.

**Правило кнопок:** акцент **не** на поверхні звичайної кнопки. Primary = `scuro`/білий
(нейтральний, контрастний); акцентна заливка кнопки лише як **подія** (підтвердити incasso = verde).

## 3. Елевація та роздільність (без обводок, без glow)

Роздільність дають **три інструменти разом**, не обводки:
- **Сходинка поверхні:** `fondo` < `superficie` (білий світліший за полотно) → картка вже видно.
- **М'яка цілеспрямована тінь** (тільки для того, що «летить»):
  | Токен | Значення | Для чого |
  |---|---|---|
  | `e1` | `0 1px 2px rgba(20,28,46,.05), 0 8px 20px -8px rgba(20,28,46,.12)` | звичайна картка |
  | `e2` | `0 14px 40px -12px rgba(20,28,46,.18)` | hero / плаваюче (nav, FAB) |
  | `e3` | `0 -10px 44px rgba(20,28,46,.14)` | оверлей-лист (sheet) знизу |
- **Внутрішня заливка-градієнт** дає глибину без рамки.

**Оверлеї (модалки/листи) — піднесення над фоном:** sheet виїжджає знизу з `e3` + затемнення-
підкладка (`scrim` чорний 35–55%, опційно `backdrop-blur` 2–8px) → екран під ним **явно нижче**.
Так «накладення віконець» має піднесення й роздільність. Гласморфізм (`backdrop-blur`+напівпрозорість)
— **тільки для оверлеїв** над вівідним фоном, не для звичайних карток на світлі.

❌ **Анти-патерни:** 1px-обводки скрізь · glow/підсвічення під блоками · однакові сірі тіні на
всьому · монотонний суцільний екран · колір на поверхні кнопок.

---

## 4. Типографіка

- **Inter Variable** — усе: display (700, від'ємний трекінг `-0.02em`) для чисел-героїв і
  заголовків (28–56px); titoli секцій 500/600; тіло 400/500; мітки UI — `+0.02em` трекінг.
- **JetBrains Mono** — дані: `codice parlante`, відсотки, дати, час, технічні мітки.
- Шкала: `56` число-герой · `28–32` заголовок екрана · `17–20` секція · `15` voce · `13` тіло · `11` мікро.

## 5. Форми

Радіуси **великі, м'які**: hero `28–32px` · картка `24px` · інпут/мала-картка `16px` · чіп `12px` ·
кнопка/пігулка `∞`. Без гострих кутів, без жорстких рамок.

## 6. Бібліотека компонентів — з РІЗНОМАНІТТЯМ

### Верхні блоки (hero / зведена інформація)
- **Carta-banca (hero-summary):** вівідна заливка (mesh/градієнт) + власний **eyebrow-заголовок**
  + головне число/ідентичність (живе при введенні) + бейдж-параметр у куті. *Як баланс-картка банку.*
- **KPI-bento:** 2–3 колір-кодовані плитки (Guadagnato/Incassato/Da incassare) — різні тінти, різні ваги.
- **Collapsing header:** при скролі стискається, число лишається.

### Списки / історія (нижні блоки)
- **Riga-transazione:** `[іконка-кружок кольору категорії] назва + мета … [сума кольору стану]`,
  згруповані за датою (sticky date-badge). Свайп-дія (Riscuoti) на потрібних.
- **Riga compatta** для «закритих» (pagato) — менша вага, спокій.
- **Voce-dossier:** аватар + назва + `codice` + мікро-сума (анаграфіка).

### Картки за станом / категорією (bento)
Колір-кодовані, **внутрішня велатура** + іконка-кружок + штамп стану. Форма/щільність ≠ між
станами (programmato — повітряна/пунктир, da incassare — гучна з великою сумою, pagato — компактна).

### Кнопки — РІЗНІ за роллю й розміром
- **Primaria** (нейтральна: `scuro`/біла пігулка) — одна головна дія.
- **Accento-evento** (`bg verde/blu/corallo`) — лише коли дія = подія (підтвердити/видалити).
- **Secondaria** (тональна `superficie-2`) · **Ghost** (текст) · **Icona** (кругла) ·
  **FAB** (плаваюча) · **Action-row** (кругла іконка + підпис ×N) · **Pill-filtri** (чіп-таби).

### Параметри введення — РІЗНІ за суттю (не «просто рядки»)
- **Numero-grande:** велике число + одиниця (тариф, сума) — окрема панель.
- **Segmented:** 2–4 взаємовиключні (persona/azienda, ore/preventivo) — повзунок.
- **Mini-chip що розгортається:** дрібні факультативні (telefono/luogo/email) в **один ряд**,
  «+» розкриває поле. *Економія простору.*
- **Stepper / Toggle / Selector-row** (аватар + назва + chevron → відкриває picker).
- **Time:** arrivo → uscita **авто-розрахунок**.

### Оверлеї (за ієрархією ваги — кожен свій тон/рух)
`info` (дрібний, нейтральний) · `dettaglio` (досьє, spring знизу) · `azione` (mesh-шапка
verde/blu, Riempimento+StampVivo) · `creazione` (фокус) · `pericolo` (corallo, центр-іконка).
Усі — `e3` + scrim, drag-handle, ESC/тап-підкладка.

### Дрібне
Чіпи-стани (`StatePill`) · аватари (тоновані; `storico` — mesh-кільце) · `codice`-чіп ·
прогрес-смуги/кільця · іконки `lucide` (stroke 2, активне 2.5; колір-кодування за категорією).

---

## 7. Композиція екрана (патерн)

Згори вниз — **4 регістри** (canone 06): **TESTATA** (hero-summary) → **RIEPILOGO**
(bento KPI) → **ELENCO** (списки/історія за датою/розділом) → **AZIONE** (nav+FAB).
*Верх = головна зведена сума/ідентичність; низ = рядки історії.* Між регістрами — повітря.

## 8. Рух (мікроінтеракції — дають життя)
NumberFlow на числах · Riempimento+StampVivo на грошах · list stagger reveal · swipe-дії ·
press-scale `.98` · spring-листи · колір-морф фону між екранами · shared-element FAB→creazione.
Усе під `prefers-reduced-motion`.

## 9. Як це «відчувається» (резюме смаку)
Білі/світлі поверхні **летять** над спокійним полотном на м'якій тіні; колір **тонко заливає**
картки за змістом; верхній блок — вівідна «карта» з головним числом; нижче — чисті колір-кодовані
рядки історії; оверлеї **піднімаються** над фоном; кнопки нейтральні, акцент — у даних; рух
дрібний і доречний. Різноманітно, але з однієї родини. Молодо, стильно, читабельно.
